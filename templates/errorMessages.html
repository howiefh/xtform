Custom Error Messages

XtForm provides multiple ways to set the error message displayed for a given error condition. In many cases the error message will not need to be specified for each element as XtForm will generate common messages for you.

Errors per element

To declare a custom error message all you need to do is declare the message on an attribute of the ng-model element. See this example below which overrides the default error messages with ones particular to this element. 


<input name="number" type="number" ng-model="model.number" class="form-control" id="num"
       required min="0" max="6" placeholder="Required & must be between 0 and 6"
       msg-max="Woah... too high!!"
       msg-min="Needs to be bigger than that!"
       msg-required="You need to fill this one out"
       msg-number="What the? I can't read this!">

App wide defaults

Most error messages will not need to be explicitly set on the input itself. XtForm provides a set of default error messages for standard error keys. These can be overridden in the configuration provider shown below.

Key Messages
minlength 'Needs to be at least {{ngMinlength}} characters long'
maxlength 'Can be no longer than {{ngMaxlength}} characters long'
required 'This field is required'
number 'Must be a number'
min 'Must be at least {{ngMin}}'
max 'Must be no greater than {{ngMax}}'
email 'Must be a valid E-mail address'
pattern 'Illegal value'
url 'Must be a valid URL'
date 'Must be a valid date'
datetimelocal 'Must be a valid date'
time 'Must be a valid time'
week 'Must be a valid week'
month 'Must be a valid month'

The error messages are interpolated using the attributes of the element. This means you have access to variables such as max/min, etc inside of the default error messages.

The following code snippet shows how to override the built in error messages to change XtForm defaults, or add your own. 


angular.module('sample').config(function (xtFormConfigProvider) {

    // Override default error messages
    xtFormConfigProvider.setErrorMessages({
        required 'Yo!, this will override the required message',
        someOtherKey: 'Here is a message for a custom validation directive'
    });
});
