Validation Strategies

XtForm introduces the concept of a "validation strategy" which defines when to show/hide validation messages to the user. An example of a strategy could be "when the element is dirty and focused". Strategies remove this logic from the view itself allowing the framework to decide when to show/hide messages.

Inbuilt Strategies

XtForm provides a few commonly used strategies out of the box.

Strategy
submitted Only if form has been submitted
dirty Only if the element in dirty
dirtyOrSubmitted Only if the element is dirty or form submitted
focusedAndDirtyOrSubmitted Both focused and (dirty or submitted)
dirtyAndFocusedOrSubmitted Both (dirty and focused) or submitted

Custom Strategies

There are configuration methods for creating custom strategies 


angular.module('sample').config(function (xtFormConfigProvider) {

    // Add custom validation strategy
    xtFormConfigProvider.addValidationStrategy('customStrategy', function (form, ngModel) {
        return ngModel.$invalid && (form.$submitted || ngModel.$dirty);
    });
});


<form strategy="customStrategy">
    ...
</form>

// Set as default for all forms
angular.module('sample').config(function (xtFormConfigProvider) {

    // Add custom validation strategy
    xtFormConfigProvider.setDefaultValidationStrategy('customStrategy');
});
