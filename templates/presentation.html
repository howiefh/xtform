Inline Validation

Basic inline form validation. Username is required, and email must be a valid email. 


<form name="inlineForm" xt-form ng-submit="save(inlineForm)" role="form" novalidate>

    <label for="inlineUsername">Username</label>
    <input name="username" type="text" class="form-control"
        id="inlineUsername" placeholder="Required field"
        ng-model="model.username" required>
    <xt-validation-inline for="inlineUsername"></xt-validation-inline>

    <label for="inlineEmail">Email</label>
    <input name="email" type="email" ng-model="model.email"
        class="form-control" id="inlineEmail"
        placeholder="Must be valid email" required>
    <xt-validation-inline for="inlineEmail"></xt-validation-inline>

    <label for="inlineNumber">Number</label>
    <input name="number" type="number" ng-model="model.number"
        class="form-control" id="inlineNumber"
        placeholder="Must be between 0 and 6" min="0" max="6">
    <xt-validation-inline for="inlineNumber"></xt-validation-inline>

    <button type="submit" class="btn btn-primary">Save</button>
</form>

Validation summary

XtForm comes with a built-in validation summary directive which lists all validation errors in one control 


<form xt-form name="form2" class="form" role="form" ng-submit="save(form2)" novalidate>

    <label for="username2">Username</label>
    <input name="username" type="text" class="form-control" id="username2"
        ng-model="model.username" required>

    <label for="email2">Email</label>
    <input name="email" type="email" ng-model="model.email"
        class="form-control" id="email2">

    <label for="number2">Number</label>
    <input name="number" type="number" ng-model="model.number"
        class="form-control" id="number2"
        placeholder="Must be between 0 and 4" min="0" max="4">

    <xt-validation-summary></xt-validation-summary>
    <button type="submit" class="btn btn-primary">Save</button>
</form>

Tooltip Validation

Standard

XtForm supports tooltip validations when integrated with the twitter bootstrap tooltip plugin or equivalent $.fn.tooltip() plugin. The following example uses the 'focusedAndDirtyOrSubmitted' strategy with focusError option set. These options ensure that the message is only shown when focused and will auto-focus on submit. For more information on strategies and other form options see Strategies. 

<form xt-form strategy="focusedAndDirtyOrSubmitted" focus-error="true"
    name="tooltipForm" role="form" ng-submit="save(tooltipForm)"
    novalidate>

    <label for="u">Username</label>
    <input name="username" id="u" ng-model="model.username"
        xt-validation-tooltip required>

    <label for="e">Email</label>
    <input name="email" id="e" type="email" ng-model="model.email"
         xt-validation-tooltip>

    <label for="n">Number</label>
    <input name="number" type="number" ng-model="model.number"
        id="n" xt-validation-tooltip placeholder="Must be between 0 and 4"
        min="0" max="4">

    <button type="submit" class="btn btn-primary">Save</button>
</form>

Hover

The tooltip trigger can be overridden on the form element by setting the tooltip-trigger attribute. 

<form xt-form strategy="invalid" focus-error="true"
    name="tooltipForm" role="form" ng-submit="save(tooltipForm)"
    novalidate tooltip-trigger="hover manual">

    <label for="u">Username</label>
    <input name="username" id="u" ng-model="model.username"
        xt-validation-tooltip required>

    <label for="e">Email</label>
    <input name="email" id="e" type="email" ng-model="model.email"
         xt-validation-tooltip>

    <label for="n">Number</label>
    <input name="number" type="number" ng-model="model.number"
        id="n" xt-validation-tooltip placeholder="Must be between 0 and 4"
        min="0" max="4">

    <button type="submit" class="btn btn-primary">Save</button>
</form>
